# 🧠 MEMORIA DEL PROYECTO: ANTIGRAVITY VOICE-AGENT ENGINE

## 📌 1. DEFINICIÓN DEL PRODUCTO

Estamos construyendo un sistema de **Landing Pages Premium para Negocios Locales** (e.g., estética, salud, servicios de lujo) con una integración nativa de **Agentes de Voz de Retell AI**. La propuesta de valor es ofrecer un sitio web de altísima calidad visual donde el cliente pueda interactuar directamente con una asistente virtual (ej. Isabela) para dudas o agendamientos.

---

## 🛠️ 2. STACK TÉCNICO REUTILIZABLE

- **Estructura**: Dockerizada (lista para Dokploy/DigitalOcean).
- **Backend (Node.js)**:
  - Gestión segura de `RETELL_API_KEY`.
  - Endpoint `/api/create-retell-call` para tokens.
  - Servidor de archivos estáticos (`frontend/dist`).
- **Frontend (Vanilla HTML + JS + Vite)**:
  - SDK de Retell para llamadas WebRTC.
  - GSAP para micro-animaciones premium.
  - Lucide para iconografía.
- **Contenerización (Dockerfile)**: Multi-stage build optimizado para Alpine con dependencias nativas instaladas (`lightningcss`, `@rollup/rollup-linux-x64-musl`).

---

## 🎨 3. ESTÁNDARES ESTÉTICOS (EL "FEEL" DEL PRODUCTO)

1. **Premium Hero**: Tipografía moderna, imagen principal impactante e inamovible por defecto.
2. **Dashboard de Estatus**: Listado interactivo de técnicas y estatus "en vivo".
3. **Galería Progresiva**: Quadrícula asimétrica con hover dinámico.
4. **Isabela Interface**:
   - **Avatar de IA**: Rostro realista generado por IA según el nicho.
   - **Modal Floating**: Ventana emergente (modal) de bienvenida antes de la llamada.
5. **Cursor Sutil**: Puntero nativo + punto dorado de seguimiento.

---

## 🔄 4. PROCESO DE ADAPTACIÓN PARA NUEVOS PROYECTOS

Al replicar esto en un nuevo cliente:

1. **Ingesta**: Leer `INFORME_NEGOCIO.txt` (servicios, precios, tono).
2. **Media**: Reemplazar `/assets/images/` y `/assets/videos/`.
3. **Voz**:
    - Actualizar `agent_id` en el frontend.
    - Personalizar textos del modal de Isabela.
4. **Grid**: Configurar los servicios reales en la sección `Stack de Técnicas`.
5. **Despliegue**: `npm run build` -> Git Push -> Dokploy.

## 🎯 6. LINEAMIENTOS ESTRATÉGICOS DE BRANDING

1. **Identidad de Agente**: Se mantendrá el nombre **"Isabela"** y su rostro oficial para todas las propuestas iniciales/demos. Solo se modificará si el cliente acepta la propuesta y solicita un cambio de identidad específico.
2. **Gestión de Logos**: En caso de que el cliente no disponga de uno, se diseñará un logo temporal elegante mediante **CSS/SVG** o se generará uno profesional con **IA**, priorizando siempre la coherencia visual con el nicho de negocio.
3. **Variación Cromática**: Se realizarán ligeros cambios en las paletas de colores para cada proyecto para evitar duplicidad exacta, pero siempre manteniendo un estándar **"Premium Lujo"** basado en la combinación obligatoria de **Dorado**, colores claros/crema y un color de acento profundo (ej. Verde Deep, Borgoña, Azul Medianoche).
